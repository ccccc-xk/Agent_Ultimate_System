# 项目五：企业级高级 RAG 知识库 — 总结文档

## 功能概述

企业知识库问答系统，支持 ES + Milvus 混合检索 + Reranker 重排序。适用于规章制度、合同规范等企业文档的智能问答。

- **混合检索**：BM25 关键词检索（Elasticsearch）+ 语义向量检索（Milvus），双路召回
- **Reranker 重排**：对召回结果进行精排，提高相关性
- **企业文档问答**：输入"员工年假有几天"，从规章制度文档中检索并生成回答
- **自动导入**：首次启动自动将 `data/` 目录下的文档导入 ES

## 技术栈

| 组件 | 技术 |
|------|------|
| 后端 | FastAPI + LangChain |
| 关键词检索 | Elasticsearch (Docker, 端口 9200) |
| 向量检索 | Milvus (Docker, 端口 19530) |
| Reranker | BGE-Reranker-v2-m3 (本地/远程) |
| Embedding | BGE-large-zh-v1.5 (本地加载) |
| LLM | Ollama qwen2.5:1.5b (端口 11434) |
| 文档 | 公司规章制度手册、合同管理规范 |

## 启动方法

```bash
cd 05_企业级高级RAG知识库
double-click start.bat
```

**依赖服务**：
- Ollama (:11434) — 已随系统启动
- Elasticsearch (Docker, :9200)

## 检索流程

```
用户问题
  ↓
BM25 检索 (ES) ──→ 候选文档集 A
  ↓
向量检索 (Milvus) ──→ 候选文档集 B
  ↓
合并去重 (A ∪ B)
  ↓
Reranker 精排
  ↓
Top-K 结果 → LLM 生成回答
```

## 注意事项

- 首次启动会自动导入 `data/` 目录下的文档到 Elasticsearch
- 企业文档放在 `backend/data/` 目录下，支持 Markdown 格式
- ES 和 Milvus 需要通过 Docker 单独启动
- Reranker 服务可选部署，未部署时降级为纯向量检索
