# 模块说明

## 核心模块

### bm25_search.py — BM25 关键词检索

| 项 | 说明 |
|-----|------|
| 职责 | Elasticsearch 全文检索，使用 IK 中文分词 |
| 核心函数 | `bm25_search(query, top_k)` → `List[dict]` |
| 索引配置 | `ik_max_word` 索引分词，`ik_smart` 搜索分词 |
| 返回格式 | `{doc_name, chunk_text, page_num, score}` |
| 依赖 | elasticsearch-py |

### vector_search.py — 向量语义检索

| 项 | 说明 |
|-----|------|
| 职责 | Milvus 向量检索，复用 BGE-large-zh Embedding |
| 核心函数 | `vector_search(query, top_k)` → `List[dict]` |
| 索引类型 | IVF_FLAT, COSINE 相似度, nlist=128, nprobe=16 |
| 返回格式 | `{doc_name, chunk_text, page_num, score}` |
| 依赖 | pymilvus, sentence-transformers |

### hybrid_search.py — 双路混合检索

| 项 | 说明 |
|-----|------|
| 职责 | 并发执行 BM25 + 向量检索，加权 RRF 融合 |
| 核心函数 | `hybrid_search(query, top_k, bm25_weight, vector_weight)` → `List[dict]` |
| RRF 公式 | `score = Σ weight_i / (k + rank_i)`, k=60 |
| 默认权重 | BM25:0.4, 向量:0.6 |
| 并发策略 | ThreadPoolExecutor 双线程并行 |

### reranker.py — Rerank 二次重排

| 项 | 说明 |
|-----|------|
| 职责 | 调用 bge-reranker-large 精排候选文档 |
| 核心函数 | `rerank(query, candidates, top_n, threshold)` → `List[dict]` |
| 服务地址 | XInference Docker, 端口 9997 |
| 阈值 | 默认 0.6，低于此分数的候选被过滤 |
| 容错 | Reranker 不可用时，回退为原始分数排序 |

### compressor.py — Context 压缩

| 项 | 说明 |
|-----|------|
| 职责 | 清洗、去重、压缩检索上下文，优化 Token 消耗 |
| 核心函数 | `compress_context(chunks)` → `(text, CompressionReport)` |
| 压缩步骤 | HTML 清洗 → 空白清理 → 冗余去除 → 句子去重 |
| Token 统计 | 使用 tiktoken (cl100k_base) |
| 目标压缩率 | > 40% |

### rag_pipeline.py — RAG 全链路

| 项 | 说明 |
|-----|------|
| 职责 | 串联检索→重排→压缩→生成的完整流程 |
| 核心函数 | `generate_answer(question, top_k)` → `dict` |
| 返回格式 | `{answer, sources, confidence, tokens_used, elapsed_ms}` |
| LangSmith | 通过环境变量启用全链路自动 Tracing |

## 基础设施模块

### embedding.py — Embedding 模型
- 单例模式，首次调用时加载 BGE-large-zh-v1.5
- HuggingFace 缓存指向 D 盘 `.cache` 目录
- 输出 1024 维归一化向量

### milvus_client.py — Milvus 客户端
- 封装 collection 创建、插入、搜索、删除操作
- 自动管理连接和索引

### llm.py — LLM 客户端
- LangChain ChatOpenAI 封装
- 支持同步和流式两种模式

### loaders/document_loader.py — 文档加载器
- 支持 PDF (PyMuPDF)、Markdown、TXT

### splitter/text_splitter.py — 文本分块器
- 中文优化的递归分割策略
- 默认 500 字符/块，50 字符重叠

## API 模块

### api/rag.py — REST 接口
- `POST /api/rag/query` — 查询接口
- `GET /api/rag/health` — 健康检查

### models/schemas.py — 数据模型
- Pydantic 请求/响应模型定义

## 脚本

### scripts/import_to_es.py — ES 数据导入
### scripts/import_to_milvus.py — Milvus 数据导入
### eval/evaluate.py — 评估脚本
### deploy/reranker/init_reranker.py — Reranker 模型初始化
