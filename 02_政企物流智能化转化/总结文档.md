# 项目二：政企物流智能化转化 — 总结文档

## 功能概述

让传统业务接口"听懂大白话"的 NLP 中间件。核心能力：

- **智能派单**：用户用口语描述故障/报修场景，系统自动解析并创建工单（如"3号楼202房间电风扇坏了" → 自动派单到维修部门）
- **大白话查数据库**：用户用自然语言提问，LLM 自动转 SQL 执行并返回结果（如"上个月有多少个未处理工单"）
- **工单管理**：工单 CRUD、状态流转、历史查询
- **查询历史**：记录每次 NLP 查询的输入和结果

## 技术栈

| 组件 | 技术 |
|------|------|
| 后端 | Spring Boot 2.7 + MyBatis-Plus |
| 前端 | Vue 3 + Vite |
| 数据库 | MySQL 8.x (端口 3306, 库名 `logistics_db`) |
| NLP | LLM 转 SQL + 规则引擎派单 |

## 接口列表

> 本项目接口无需认证，可直接调用。

### NLP 智能接口

| 方法 | URL | 功能 | 参数 | 示例 |
|------|-----|------|------|------|
| POST | `/api/nlp/dispatch` | 口语化智能派单 | `{"text": "口语描述"}` | `{"text":"3号楼202房间电风扇坏了"}` |
| POST | `/api/nlp/query` | 大白话查数据库 | `{"question": "自然语言问题"}` | `{"question":"上个月有多少工单"}` |
| GET | `/api/nlp/query/history` | 查询历史记录 | `pageNum`, `pageSize` | — |

### 工单管理接口

| 方法 | URL | 功能 | 参数 |
|------|-----|------|------|
| GET | `/api/work-order/list` | 工单分页列表 | `pageNum`, `pageSize`, `status` |
| GET | `/api/work-order/{id}` | 工单详情 | path: id |
| PUT | `/api/work-order/status` | 更新工单状态 | `{"id": 1, "status": "completed"}` |
| PUT | `/api/work-order` | 更新工单详情 | JSON body |
| DELETE | `/api/work-order/{id}` | 删除工单 | path: id |

### 快速测试

```bash
# 测试智能派单
curl -X POST http://localhost:9002/api/nlp/dispatch \
  -H "Content-Type: application/json" \
  -d '{"text":"3号楼202房间电风扇坏了"}'

# 测试大白话查库
curl -X POST http://localhost:9002/api/nlp/query \
  -H "Content-Type: application/json" \
  -d '{"question":"上个月有多少个未处理工单"}'
```

### 响应格式

```json
{
  "code": 200,
  "message": "success",
  "data": { ... }
}
```

## 注意事项

- 智能派单接口会自动解析口语文本中的地址、故障类型等信息
- 大白话查库接口依赖 LLM 将自然语言转为 SQL，复杂查询可能需要几秒
- 工单状态枚举：`pending`(待处理) / `processing`(处理中) / `completed`(已完成)
