# 项目一：企业管理基座系统 — 总结文档

## 功能概述

企业管理系统的后端基座，提供企业级后台管理所需的核心能力：

- **RBAC 权限管理**：用户、角色、菜单三级权限体系，JWT Token 认证
- **财务报表管理**：财务数据 CRUD、分页查询、Excel 导入导出
- **AI 智能财务摘要**：输入月份，AI 自动生成财务分析摘要
- **用户管理**：用户增删改查、密码重置
- **仪表盘**：财务数据汇总统计

## 技术栈

| 组件 | 技术 |
|------|------|
| 后端 | Spring Boot 2.7 + MyBatis-Plus |
| 前端 | Vue 3 + Vite + Element Plus + ECharts |
| 数据库 | MySQL 8.x (端口 3306, 库名 `enterprise_db`) |
| 认证 | JWT Token (24小时过期) |

## 项目页面地址

| 页面 | 地址 | 说明 |
|------|------|------|
| **前端管理后台** | [http://localhost:5173](http://localhost:5173) | Vue 管理后台，登录后可操作 |
| **后端 API** | [http://localhost:9001](http://localhost:9001) | Spring Boot 后端服务 |

## 接口列表

### 认证接口

获取 Token（调用其他接口前必须先获取）：
```bash
curl -X POST http://localhost:9001/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username":"admin","password":"123456"}'
```

### 业务接口

| 方法 | URL | 功能 | 参数 | 认证 |
|------|-----|------|------|------|
| POST | `/api/auth/login` | 登录获取Token | `{username, password}` | 无 |
| GET | `/api/auth/info` | 当前用户信息 | — | JWT |
| GET | `/api/dashboard/summary` | 仪表盘汇总数据 | — | JWT |
| GET | `/api/finance/page` | 财务分页查询 | `month`, `pageNum`, `pageSize` | JWT |
| POST | `/api/finance/ai-summary` | AI财务摘要 | `month` (query参数) | JWT |
| POST | `/api/finance/import` | Excel导入财务数据 | multipart file | JWT |
| GET | `/api/finance/template` | 下载导入模板 | — | JWT |
| GET | `/api/user/page` | 用户分页搜索 | `keyword`, `pageNum`, `pageSize` | JWT |
| POST | `/api/user` | 新增用户 | JSON body | JWT |
| PUT | `/api/user` | 更新用户 | JSON body | JWT |
| DELETE | `/api/user/{id}` | 删除用户 | path: id | JWT |
| PUT | `/api/user/reset-password/{id}` | 重置密码 | path: id | JWT |
| GET | `/api/role/page` | 角色分页查询 | `pageNum`, `pageSize` | JWT |
| GET | `/api/menu/list` | 当前用户菜单 | — | JWT |

### 响应格式

```json
{
  "code": 200,
  "msg": "success",
  "data": { "records": [...], "total": 100 }
}
```

## 注意事项

- JWT Token 有效期 24 小时，过期后需重新登录获取
- 调用受保护接口时在 Header 中添加 `Authorization: Bearer <token>`
- 前端 vite.config.js 代理已配置为 `localhost:9001`，前后端联调无需额外配置
