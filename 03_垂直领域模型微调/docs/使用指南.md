## 这个项目是干什么的？

公共大模型（如 Qwen、DeepSeek）回答问题时格式不稳定——有时输出 JSON，有时夹带解释文字。

**这个项目通过微调，让模型学会"守规矩"**：给它一个任务，它 100% 返回标准格式，不会乱写。

训练好的模型会变成一个 **API 服务**，其他项目（04医疗问诊、05知识库、06Agent）调用这个 API 就能获得格式稳定的输出。

---

## 第 2 步：在云 GPU 服务器上训练模型

### 2.1 租一台 GPU 服务器

推荐平台（按价格排序）：

| 平台 | GPU | 显存 | 大约价格 | 链接 |
|------|-----|------|----------|------|
| AutoDL | RTX 4090 | 24GB | ≈2元/小时 | autodl.com |

- CUDA 版本选 **12.1 或更高**

### 2.4 开始训练

```bash
# 启动训练（选 Qwen2.5，中文效果更好）
bash train.sh qwen25
```

训练过程中你会看到：
- `loss` 值不断下降（正常现象）
- 每 50 步保存一次 checkpoint
- 全程约 30-60 分钟

**训练完成后**，模型 LoRA 权重保存在 `output/qwen25_lora/` 目录。

### 2.5 验证训练效果

```bash
# 合并 LoRA 权重到基座模型
python scripts/merge_lora.py --lora_path output/qwen25_lora --output_path models/merged_model

# 运行合规率验证
python scripts/validate_model.py --model_path models/merged_model
```

验证会输出类似：
```
总测试数: 20
通过:     20
失败:     0
合规率:   100.00%
```

**如果合规率 ≥ 95%**：训练成功，可以部署。
**如果合规率 < 95%**：查看 `docs/validation_report.json` 的失败分析，补充数据重新训练。

### 2.6 部署为 API 服务

```bash
# 启动 vLLM API 服务
python -m vllm.entrypoints.openai.api_server \
    --model models/merged_model \
    --served-model-name vertical-sft-model \
    --max-model-len 4096 \
    --gpu-memory-utilization 0.9 \
    --port 8000 \
    --api-key your-secret-key \
    --trust-remote-code
```

服务启动后，你的服务器就变成了一个 API 端点：
- 地址：`http://你的服务器IP:8000`
- API Key：`your-secret-key`

### 2.7 测试 API

在你的 Windows 电脑上测试：

```bash
curl http://你的服务器IP:8000/v1/chat/completions ^
  -H "Authorization: Bearer your-secret-key" ^
  -H "Content-Type: application/json" ^
  -d "{\"model\":\"vertical-sft-model\",\"messages\":[{\"role\":\"user\",\"content\":\"分析投诉文本，归类标签并提取诉求。衣服线头全开了，退钱！\"}]}"
```

```python

    base_url="http://你的服务器IP:8000/v1",

# 测试投诉分类

    model="vertical-sft-model",
```

## 常见问题

**Q: 训练时显存不够怎么办？**
降低 batch size：在 yaml 配置里把 `per_device_train_batch_size` 改成 2。

**Q: 训练 loss 不下降？**
检查数据格式，降低学习率到 1e-4。

**Q: API 服务启动后访问不了？**
检查服务器安全组是否开放了 8000 端口。

**Q: 想重新训练，怎么重来？**
删掉 `output/qwen25_lora/` 目录，重新 `bash train.sh qwen25`。
