# 04/05/06 项目对接本地模型指南

> 本文档说明如何让项目 04（医疗问诊）、05（知识库）、06（Agent）连接到本地 Qwen2.5-1.5B 模型。

---

## 前置条件

本地模型已通过 Ollama 运行，地址为：

```
http://127.0.0.1:11434/v1
```

启动方式（如果模型没运行）：

```bash
# Windows
"C:\Users\Administrator\AppData\Local\Programs\Ollama\ollama app.exe"

# 然后确认模型存在
"C:\Users\Administrator\AppData\Local\Programs\Ollama\ollama.exe" list
# 应显示 qwen2.5:1.5b
```

---

## 对接方式

项目 04 和 05 都通过 `.env` 文件配置 LLM 连接参数。只需修改三个变量：

| 变量 | 原值（云端 DashScope） | 改为（本地 Ollama） |
|------|----------------------|-------------------|
| `OPENAI_API_KEY` | `sk-xxx`（你的 DashScope key） | `ollama`（任意值，本地不需要真 key） |
| `OPENAI_BASE_URL` | `https://dashscope.aliyuncs.com/compatible-mode/v1` | `http://127.0.0.1:11434/v1` |
| `LLM_MODEL_NAME` | `qwen-plus` | `qwen2.5:1.5b` |

---

## 项目 04（医疗智能问诊）具体操作

1. 打开 `04_医疗智能问诊基础RAG/.env`（没有就复制 `.env.example`）
2. 修改为：

```
OPENAI_API_KEY=ollama
OPENAI_BASE_URL=http://127.0.0.1:11434/v1
LLM_MODEL_NAME=qwen2.5:1.5b
EMBEDDING_MODEL_PATH=BAAI/bge-large-zh-v1.5
```

3. 重启后端服务即可生效

---

## 项目 05（企业级 RAG 知识库）具体操作

1. 打开 `05_企业级高级RAG知识库/.env`（没有就根据 `配置参数说明.md` 创建）
2. LLM 部分修改为：

```
OPENAI_API_KEY=ollama
OPENAI_BASE_URL=http://127.0.0.1:11434/v1
LLM_MODEL_NAME=qwen2.5:1.5b
```

3. 其他配置（Milvus、Elasticsearch、Embedding、Reranker）保持不变
4. 重启后端服务

---

## 项目 06（Agent 智能体）具体操作

项目 06 调用 04/05 的能力，本身不直接连 LLM。只要 04/05 正常运行，06 自动可用。

如果 06 有自己的 LLM 配置（如 prompts 目录下的配置），同样改为：

```
base_url: http://127.0.0.1:11434/v1
api_key: ollama
model: qwen2.5:1.5b
```

---

## 快速验证

修改完 `.env` 后，用以下 Python 脚本验证连接是否正常：

```python
from openai import OpenAI

client = OpenAI(
    base_url="http://127.0.0.1:11434/v1",
    api_key="ollama"
)

response = client.chat.completions.create(
    model="qwen2.5:1.5b",
    messages=[{"role": "user", "content": "你好，请简单介绍一下自己"}],
    temperature=0.1
)
print(response.choices[0].message.content)
```

如果返回正常中文回复，说明对接成功。

---

## 注意事项

1. **模型能力有限**：Qwen2.5-1.5B 是最小的模型，复杂推理和格式控制不如 7B/14B
2. **输出格式不稳定**：模型可能在 JSON 输出中夹带解释文字，后续项目如需严格格式，需要加后处理
3. **响应速度**：本地 CPU 推理较慢（每条 2-10 秒），GPU 推理会快很多
4. **端口占用**：Ollama 占用 11434 端口，确保没有其他服务冲突
5. **升级路径**：后续如需更好的效果，可以：
   - 换更大模型：`ollama pull qwen2.5:7b`（需要 8GB+ 显存）
   - 回到云端 API：把 `.env` 改回 DashScope 配置
   - 完成微调后部署：用项目 03 的训练产物替换

---

## 切换回云端 API

如果本地模型效果不够，随时可以切回 DashScope 云端：

```
OPENAI_API_KEY=sk-你的真实key
OPENAI_BASE_URL=https://dashscope.aliyuncs.com/compatible-mode/v1
LLM_MODEL_NAME=qwen-plus
```

重启服务即可。
