# 项目四：医疗智能问诊基础 RAG — 总结文档

## 功能概述

基于 Milvus 向量检索的医疗知识库问答系统。用户输入疾病/症状/药物问题，系统从医疗文档中检索相关片段，用 LLM 生成专业回答。

- **医疗问答**：输入"感冒了吃什么药"，从医疗文档中检索相关药物说明，生成专业回答
- **流式输出**：支持 SSE 流式返回，逐字显示回答
- **知识库管理**：上传/查看/删除医疗文档
- **向量检索**：基于 BGE Embedding + Milvus 的语义搜索

## 技术栈

| 组件 | 技术 |
|------|------|
| 后端 | FastAPI + LangChain |
| 向量库 | Milvus (Docker, 端口 19530) |
| Embedding | BGE-large-zh-v1.5 (本地加载) |
| LLM | Ollama qwen2.5:1.5b (端口 11434) |
| 文档 | 医院科室介绍、常用药物说明、常见疾病诊疗指南 |

## 启动方法

```bash
cd 04_医疗智能问诊基础RAG/backend
double-click start.bat

# 首次启动会下载 embedding 模型（约 1.3GB），后续秒启
```

**依赖服务**：
- Ollama (:11434) — 已随系统启动
- Milvus (Docker, :19530) — 需通过 Docker 启动

## 接口列表

> 本项目接口无需认证，可直接调用。

| 方法 | URL | 功能 | 参数 |
|------|-----|------|------|
| POST | `/api/chat` | 医疗知识库问答 | `{"question": "问题", "top_k": 3}` |
| POST | `/api/chat/stream` | SSE 流式医疗问答 | 同上 |
| POST | `/api/knowledge/upload` | 上传医疗文档 | multipart file |
| GET | `/api/knowledge/list` | 列出已导入文档 | — |
| DELETE | `/api/knowledge/{doc_name}` | 删除文档 | path: doc_name |
| GET | `/api/health` | 健康检查 | — |

### 快速测试

```bash
# 测试医疗问答
curl -X POST http://localhost:9004/api/chat \
  -H "Content-Type: application/json" \
  -d '{"question":"感冒了吃什么药"}'

# 健康检查
curl http://localhost:9004/api/health
```

### 响应格式

```json
{
  "answer": "根据医疗文档，感冒患者可以...",
  "sources": ["常用药物说明.md"],
  "confidence": 0.85
}
```

## 注意事项

- 首次启动需下载 BGE Embedding 模型（约 1.3GB），请确保网络畅通
- 医疗文档放在 `backend/data/` 目录下，支持 Markdown 格式
- Milvus 需要通过 Docker 单独启动：`docker run -d --name milvus -p 19530:19530 milvusdb/milvus`
