## 1. Agent 对话

### POST `/v1/chat-messages` — SSE 流式对话

Agent 应用只支持 streaming 模式。发送用户消息后，通过 SSE 事件流逐块返回 Agent 的思考过程和回答。

**请求体**：

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| inputs | object | 是 | 空对象 `{}` |
| query | string | 是 | 用户消息 |
| response_mode | string | 是 | 固定 `"streaming"` |
| user | string | 是 | 用户标识（任意字符串） |
| conversation_id | string | 否 | 对话 ID（多轮对话时传上次返回的 ID） |

**curl**：
```bash
curl -X POST http://localhost:9006/v1/chat-messages \
  -H "Authorization: Bearer app-jX3smW5hgad8jxsOhqp1osyo" \
  -H "Content-Type: application/json" \
  -d '{
    "inputs": {},
    "query": "3号楼202房间电风扇坏了",
    "response_mode": "streaming",
    "user": "user-001"
  }' --no-buffer
```

**SSE 事件流**：
```
event: message
data: {"event":"message","answer":"您好，","conversation_id":"abc123","message_id":"msg-001"}

event: message
data: {"event":"message","answer":"您反馈的电风扇故障问题，","conversation_id":"abc123"}

event: message
data: {"event":"message","answer":"我们已创建维修工单 #16。","conversation_id":"abc123"}

event: message_end
data: {"event":"message_end","conversation_id":"abc123","metadata":{"usage":{...}}}
```

**关键事件类型**：

| event | 说明 |
|-------|------|
| message | Agent 回答的文本片段 |
| agent_thought | Agent 思考过程（调用了哪个工具等） |
| message_end | 对话结束，包含 usage 信息 |

### POST `/v1/chat-messages` — 多轮对话

在请求中传入上次返回的 `conversation_id`，Agent 会自动保持上下文：

```bash
curl -X POST http://localhost:9006/v1/chat-messages \
  -H "Authorization: Bearer app-jX3smW5hgad8jxsOhqp1osyo" \
  -H "Content-Type: application/json" \
  -d '{

    "response_mode": "streaming",
    "user": "user-001",
    "conversation_id": "abc123"
```
