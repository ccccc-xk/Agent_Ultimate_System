# 企业微信 / 微信嵌入方案

## 一、企业微信 H5 应用嵌入

### 1.1 创建企业微信 H5 应用

1. 登录 [企业微信管理后台](https://work.weixin.qq.com/wework_admin/frame)
2. 应用管理 → 自建 → 创建应用
3. 填写应用名称（如"AI智能客服"）、上传 Logo
4. 设置应用主页 URL（部署聊天页面的地址）
5. 设置可见范围（需要使用的部门/人员）

### 1.2 方案 A：iframe 嵌入（推荐）

创建独立的聊天页面，通过 iframe 嵌入企业微信 H5 应用：

```html
<!DOCTYPE html>
<html lang="zh-CN">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0, user-scalable=no">
  <title>AI智能客服</title>
  <style>
    * { margin: 0; padding: 0; box-sizing: border-box; }
    body { height: 100vh; overflow: hidden; }
    iframe {
      width: 100%;
      height: 100%;
      border: none;
    }
  </style>
</head>
<body>
  <iframe src="https://your-chat-page-url.com/chat.html"
          allow="microphone; camera"></iframe>
</body>
</html>
```

### 1.3 方案 B：JSSDK 原生集成

在企业微信 H5 页面中通过 JSSDK 获取用户身份，然后嵌入聊天组件：

```html
<script src="https://res.wx.qq.com/wwopen/js/jsapi/jweixin-1.2.0.js"></script>
<script src="chat-widget.js"></script>
<script>
  // 企业微信 JSSDK 初始化
  wx.config({
    beta: true,
    debug: false,
    appId: 'YOUR_CORP_ID',
    timestamp: timestamp,
    nonceStr: nonceStr,
    signature: signature,
    jsApiList: ['agentConfig']
  });

  wx.ready(function() {
    // 初始化聊天组件
    AgentChatWidget.init({
      apiBase: 'https://your-dify-host.com',
      apiKey: 'app-your-api-key',
      primaryColor: '#1989fa',
      title: 'AI智能客服'
    });
  });
</script>
```

### 1.4 身份传递

企业微信可以传递用户身份信息（userId），将其作为 Dify Chat API 的 `user` 参数，实现：
- 用户级别的对话隔离
- 对话历史按用户存储
- 操作审计追踪

## 二、微信公众号客服消息

### 2.1 架构

```
用户发送消息 → 微信公众号 → 微信服务器
                      ↓ 回调
                你的后端服务
                      ↓
                Dify Chat API
                      ↓
                返回回复消息
                      ↓
                微信服务器 → 用户收到回复
```

### 2.2 实现步骤

1. 在 [微信公众平台](https://mp.weixin.qq.com/) 注册服务号
2. 开通客服消息功能
3. 配置服务器 URL、Token、EncodingAESKey
4. 实现后端回调接口：

```python
# Flask 示例
from flask import Flask, request
import requests

app = Flask(__name__)
DIFY_API = 'http://your-dify-host/v1/chat-messages'
DIFY_KEY = 'app-your-api-key'

@app.route('/wechat/callback', methods=['POST'])
def wechat_callback():
    # 解析微信消息
    msg = parse_wechat_message(request)
    
    # 调用 Dify
    resp = requests.post(DIFY_API, json={
        'query': msg['content'],
        'response_mode': 'blocking',
        'user': msg['from_user']
    }, headers={'Authorization': f'Bearer {DIFY_KEY}'})
    
    answer = resp.json().get('answer', '抱歉，暂时无法回答')
    
    # 返回微信消息格式
    return build_wechat_reply(msg['from_user'], answer)
```

### 2.3 注意事项
n- 公众号消息 5 秒超时，可能需要先回复"正在思考..."再异步处理
- 建议使用 Dify 的 blocking 模式，或先返回临时回复后用客服消息接口追加
- 服务号每月可发送 4 条主动推送消息（模板消息除外）

## 三、微信小程序

### 3.1 限制

- 小程序 `web-view` 组件只能打开已备案的域名
- 不支持 WebSocket 跨域连接
- SSE 长连接在小程序中可能不稳定

### 3.2 推荐方案：后端代理

不建议直接嵌入，而是通过后端代理转发：

```
小程序 → 你的后端服务 → Dify API → 返回结果
```

后端使用 blocking 模式调用 Dify，小程序通过普通 HTTP 请求获取回复。

### 3.3 替代方案

如果需要在小程序中使用 AI 客服，建议：
1. 使用小程序原生组件构建聊天 UI
2. 通过 `wx.request` 调用后端 API
3. 后端转发到 Dify Chat API（blocking 模式）
4. 对于长回复，使用轮询或 WebSocket 方案

## 四、安全建议

1. **HTTPS 必须**：生产环境必须使用 HTTPS
2. **API Key 保护**：Dify API Key 不要暴露在前端代码中（小程序场景通过后端代理）
3. **用户鉴权**：接入企业微信身份验证，避免未授权访问
4. **频率限制**：在后端添加请求频率限制，防止滥用
5. **日志审计**：记录所有对话日志，便于问题排查和合规审查
