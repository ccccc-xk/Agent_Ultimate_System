## 二、Dify 平台部署

### 2.1 克隆仓库

```bash
cd D:\Codex-project
git clone https://github.com/langgenius/dify.git
cd dify\docker
```

### 2.2 配置环境变量

复制环境变量模板：
```bash
copy .env.example .env
```

编辑 `.env`，设置关键配置：

```ini
# 安全密钥（生成方式: python -c "import secrets; print(secrets.token_hex(32))")
SECRET_KEY=your-generated-secret-key

# 初始管理员密码（首次启动时设置）
INIT_PASSWORD=your-admin-password

# 服务 URL
CONSOLE_WEB_URL=http://localhost
APP_WEB_URL=http://localhost

# 端口
EXPOSE_NGINX_PORT=80
```

### 2.3 启动 Dify

```bash
docker compose up -d
```

检查服务状态：
```bash
docker compose ps
```

预期看到以下容器均为 `Up` 或 `healthy` 状态：
- docker-api-1
- docker-web-1
- docker-worker-1
- docker-worker_beat-1
- docker-db_postgres-1
- docker-redis-1
- docker-nginx-1
- docker-sandbox-1
- docker-weaviate-1
- docker-ssrf_proxy-1
- docker-plugin_daemon-1

### 2.4 访问 Dify Web UI

浏览器打开 `http://localhost`，首次访问会进入初始化页面。

1. 创建管理员账号（或使用 INIT_PASSWORD）
2. 登录后进入主控台

**磁盘占用**：Dify 镜像总计约 4-5 GB

## 三、配置大模型供应商

### 3.1 项目三 vLLM（首选）

如果本机有 GPU 且 vLLM 已启动（端口 9004）：

1. 进入 Dify → 设置 → 模型供应商
2. 选择 "OpenAI Compatible" 或添加自定义模型
3. 配置：
   - API Base URL: `http://host.docker.internal:9004/v1`
   - API Key: `vertical-sft-secret-key`
   - Model Name: `vertical-sft-model`
4. 点击"保存"并测试连接

### 3.2 备选方案：DeepSeek

如无 GPU 或 vLLM 未就绪：

1. 注册 [DeepSeek API](https://platform.deepseek.com/)
2. 在 Dify 模型供应商中添加 DeepSeek
3. 配置 API Key
4. 选择 `deepseek-chat` 模型

### 3.3 备选方案：通义千问

1. 开通 [通义千问 API](https://dashscope.console.aliyun.com/)
2. 在 Dify 中添加 OpenAI Compatible 模型
3. API Base: `https://dashscope.aliyuncs.com/compatible-mode/v1`

## 五、导入 OpenAPI 工具

### 5.1 导入工具定义

1. 在 Dify Web UI → 工具 → 自定义 API 工具
2. 点击"导入"
3. 依次导入以下文件：
   - `openapi/project1-tools.json` — 项目一（3 个工具）
   - `openapi/project2-tools.json` — 项目二（2 个工具）
   - `openapi/project4-medical-rag.json` — 项目四（1 个工具）
   - `openapi/project5-rag.json` — 项目五（1 个工具）

### 5.2 配置认证

对于项目一的工具（需要 JWT Token）：

1. 先获取 Token：
   ```bash
   curl -X POST http://localhost:9001/api/auth/login \
     -H "Content-Type: application/json" \
     -d '{"username": "admin", "password": "123456"}'
   ```
   从返回的 `data` 字段获取 JWT Token。

2. 在 Dify 工具管理中，对项目一的三个工具分别配置：
   - 认证方式: API Key / Bearer Token
   - Token: `Bearer <your-jwt-token>`

3. Token 有效期 24 小时，过期后需重新获取并更新

### 5.3 测试工具

在 Dify 工具管理页面，对每个工具点击"测试"，输入示例参数验证连通性。

## 十、常见问题

### Q: Docker 容器启动失败，提示端口冲突
检查端口占用：
```bash
netstat -ano | findstr :9001
netstat -ano | findstr :9002
netstat -ano | findstr :8082
netstat -ano | findstr :8083
```

### Q: 工具调用超时
确认后端服务已启动且 Docker 容器能通过 `host.docker.internal` 访问宿主机。

### Q: LLM 响应很慢
如果使用远程 API（DeepSeek/通义千问），延迟属正常。如使用本地 vLLM，检查 GPU 显存是否足够。

### Q: Token 过期
重新调用项目一的登录接口获取新 Token，更新 Dify 工具配置中的 Bearer Token。

### Q: 知识库召回质量差
检查 ES 和 Milvus 是否正常运行：
```bash
curl http://localhost:9200/_cluster/health
curl http://localhost:8083/api/rag/health
curl http://localhost:8082/docs
```

### Q: 项目四和项目五的知识库有什么区别？
- **项目四（端口 8082）**: 医疗领域，包含疾病诊疗指南、药物说明、科室介绍，仅用 Milvus 向量检索
- **项目五（端口 8083）**: 企业通用领域，包含规章制度、合同管理等，用 ES+Milvus 混合检索+Rerank
