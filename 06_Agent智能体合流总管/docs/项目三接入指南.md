## 阶段一：训练模型

### 选项A：云GPU服务器（推荐）

1. 注册 AutoDL / 阿里云 / 腾讯云 GPU 实例
2. 选择 A100 40GB 或 RTX 4090 24GB
3. 上传项目三代码到服务器
4. 执行训练：
   ```bash
   # 安装依赖
   pip install -r requirements.txt
   
   # 生成训练数据
   python scripts/generate_complaint.py
   python scripts/generate_workorder.py
   python scripts/generate_sql.py
   python scripts/merge_datasets.py
   python scripts/clean_dataset.py
   python scripts/split_testset.py
   
   # 启动训练
   bash train.sh qwen25
   
   # 合并LoRA权重
   python scripts/merge_lora.py --lora_path output/qwen25_lora
   
   # 验证合规率
   python scripts/validate_model.py --model_path models/merged_model
   ```
5. 训练完成后下载 models/merged_model 到本地

### 选项B：本地量化训练（显存不足）

如果不想用云服务器，可以用量化方式在 6GB 显存上微调小模型：
- 基座模型改为 Qwen2.5-3B-Instruct（更小）
- 使用 4-bit 量化训练
- 但效果可能不如全精度训练

## 阶段二：部署 vLLM API

训练完成后，在本地启动 vLLM：

```bash
# 方式一：直接启动（需要CUDA环境）
cd 03_垂直领域模型微调
bash deploy/start_vllm.sh models/merged_model vertical-sft-secret-key

# 方式二：Docker启动
cd 03_垂直领域模型微调/deploy
docker build -t vllm-sft .
docker run -d --gpus all -p 9004:9004 \
  -v $(pwd)/../models/merged_model:/app/model \
  vllm-sft \
  --api-key vertical-sft-secret-key
```

验证部署：
```bash
# 检查模型列表
curl http://localhost:9004/v1/models

# 测试对话
curl -X POST http://localhost:9004/v1/chat/completions \
  -H "Authorization: Bearer vertical-sft-secret-key" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "vertical-sft-model",
    "messages": [
      {"role": "user", "content": "3楼空调不制冷"}
    ],
    "temperature": 0.1
  }'
```

3. 配置认证：
   - 类型: Bearer Token
   - Token: `vertical-sft-secret-key`
4. 测试工具调用

```
当遇到以下场景时，优先调用微调模型工具（而非通用大模型）：

- NL2SQL：system提示"你是一个SQL生成助手"，user传入查询问题
```

## 三大业务场景说明

### 场景一：投诉分类归因

**输入**: 用户投诉文本
**输出**: `【分类】：X\n【核心诉求】：Y`

示例：
- "衣服线头全开了，退钱！" → 【分类】：质量问题 【核心诉求】：退款退货
- "快递三天没到，耽误事" → 【分类】：物流问题 【核心诉求】：催单配送

### 场景二：口语化工单提取

**输入**: 口语化报修描述
**输出**: 严格 JSON 格式

示例：
- "3楼空调不制冷" → `{"location":"3楼","issue":"空调不制冷","priority":"普通","category":"设备维修"}`

### 场景三：自然语言转SQL

**输入**: 管理查询问题
**输出**: SELECT SQL 语句

示例：
- "上月销售额最高的产品" → `SELECT product_name, SUM(amount) as total FROM sales WHERE month='2026-05' GROUP BY product_name ORDER BY total DESC LIMIT 1`

## Token 消耗控制

微调模型运行在本地 vLLM，**不消耗 DeepSeek API 额度**，零成本调用。

建议配置：
- temperature: 0.1（微调模型需要低温度保证格式稳定）
- max_tokens: 512（足够输出分类/JSON/SQL）

## 注意事项

1. **显存要求**: vLLM 推理至少需要 8GB 显存（Qwen2.5-7B）
2. **首次加载**: vLLM 启动时加载模型需要 30-60 秒
3. **并发限制**: 单 GPU 最大并发约 4-8 个请求
4. **模型格式**: 必须是合并后的完整模型，不能直接用 LoRA 适配器
